# Cinecam 要件定義 v0.2
**複数台同期・日常シネマ体験アプリ**

---

## 1. 目的
複数台のiPhoneを同期させ、ワンボタンで同時録画を開始できる。  
撮影後、各端末に撮影データを共有し、同一時間軸上で編集可能な **"マルチアングル映像"** を生成する。  
**日常を映画化する体験を目的とする。**

---

## 2. 想定利用シーン
- キッチンでコーヒーを淹れる
- 喫茶店での会話
- 料理シーン
- 誕生日ケーキ
- 簡易対談動画
- 屋外での活動（公園、散歩など）

**前提条件:**
- 2台以上のiPhone（iOS 16.0以上）
- 各端末でWi-FiとBluetoothをオンにする
- **インターネット接続不要**
- **Wi-Fiルーター不要**（MultipeerConnectivityによるP2P通信）

---

## 3. MVP（最小構成）

### ■ 接続機能
- **MultipeerConnectivity** による自動デバイス検出
- デバイス間で直接P2P通信（アドホックネットワーク）
- 1台を **マスター**、他を **スレーブ** に設定
- 接続状態表示（接続台数、デバイス名表示）
- 通信距離: 約10m（見通し）
- 推奨接続台数: 2〜5台

### ■ 同期録画
- マスターの **RECボタン** で全端末同時録画開始
- 録画開始コマンド送信時に **timestamp** を全端末で記録
- 録画停止も同期（マスター制御）
- 目標同期精度: **±100ms以内**（会話用途で十分）

### ■ データ管理
**Phase1（MVP）:**
- 各端末にローカル保存のみ
- ファイル名に **セッションID + timestamp** 付与
- 後で手動集約（AirDrop、iCloud、Macへの転送など）

**Phase2（将来）:**
- MultipeerConnectivity経由で自動転送
- マスター端末への集約オプション
- チャンク分割による安定転送

### ■ 同期補正（Phase3）
- 録画開始timestamp差分の保存
- 音声波形による自動微調整（後処理）
- タイムコードレベルの厳密同期は不要（映像制作用途ではないため）

### ■ マルチアングル編集UI（Phase4）
- タイムライン上の青枠は**排他的セレクター**として機能する
  - 全時間軸において、ある時刻に「選択されているカメラ」は常に1台のみ
  - 選択中のシーケンスに青枠をドラッグすると、その範囲の他トラックの青枠は自動分割・削除
  - 青枠 = 書き出し動画に採用される区間
- 再生ヘッドの位置でプレビュー映像が自動切り替わる（選択されているトラックを表示）
- 書き出しはAVCompositionでセグメントを順番に結合し、1本の動画を生成

## 4. 技術仕様（iOS）

### 開発言語・フレームワーク
- **Swift**
- **SwiftUI** (UI構築)
- **AVFoundation** (カメラ・録画制御)
- **MultipeerConnectivity** (ローカル通信)
- **AVAudioSession** (音声処理)
- **CMTime / CMSampleBuffer** (timestamp取得)

### 通信プロトコル
```
MultipeerConnectivity:
├─ サービスタイプ: "cinecam-sync"
├─ コマンド形式: JSON
└─ 主要コマンド:
    ├─ "connect_request"   (接続要求)
    ├─ "start_recording"   (録画開始 + timestamp)
    ├─ "stop_recording"    (録画停止)
    └─ "session_info"      (セッションID共有)
```

### データ構造
```
録画ファイル名: 
cinecam_[SessionID]_[DeviceName]_[Timestamp].mov

例: cinecam_A3F2E_iPhone13_1708077600.mov
```

### 動作要件
- **クラウド依存なし**（スタンドアロン動作）
- オフライン完結
- バックグラウンド録画非対応（iOS制約のため）

## 5. UX設計思想

### デザインコンセプト
**「極限までシンプル。同期している感は見せない。自然に映画になる体験を優先。」**

### メイン画面（イメージ）
```
┌─────────────────────┐
│      Cinecam        │
│                     │
│   接続: 3台         │
│   ● iPhone 13       │
│   ● iPhone 12 Pro   │
│   ● iPad Air        │
│                     │
│   [ ● REC ]         │
│                     │
│   [ 設定 ]          │
└─────────────────────┘
```

### 操作フロー
1. アプリ起動 → 自動で周辺デバイス検出
2. 接続デバイスをタップで追加
3. マスター端末で **RECボタン** → 全端末同時録画開始
4. もう一度タップ → 全端末停止
5. 各端末のカメラロールに自動保存

### 設定画面（最小限）
- マスター/スレーブ切替
- デバイス名編集
- 録画解像度（1080p / 4K）
- フレームレート（30fps / 60fps）

---

## 6. 精度要件

| 項目 | 目標値 | 備考 |
|------|--------|------|
| 同期精度 | ±100ms以内 | 会話用途で十分 |
| 接続安定性 | 通信距離10m以内で安定 | MultipeerConnectivity特性 |
| 録画開始遅延 | コマンド送信後50ms以内 | ローカル通信の利点 |
| 転送速度 | 数MB/秒 | Phase2での動画転送時 |

**タイムコードレベルの厳密同期は不要**  
（プロ用映像制作ではなく、日常のシネマ体験が目的のため）

## 9. 技術的課題と対策

### 課題1: 接続安定性
- **対策:** MultipeerConnectivityの再接続ロジック実装
- **対策:** 接続監視とタイムアウト処理

### 課題2: 録画開始のタイミングずれ
- **対策:** 高精度timestamp取得（CMTime使用）
- **対策:** コマンド送信遅延の計測と補正

### 課題3: バッテリー消費
- **対策:** 録画時以外は通信を最小化
- **対策:** 低消費電力モード対応

### 課題4: ストレージ容量
- **対策:** 録画前の容量チェック
- **対策:** 警告表示の実装

### 課題5: 排他的セグメント整合性
- **対策:** `ExclusiveEditTimeline` クラスがセグメント配列の整合性を一元管理
- **対策:** ドラッグ操作のたびに `applySelection(range:for:)` を呼び出し、他トラックを自動分割
- **対策:** セグメントは常に時間軸でソート・重複なし・隙間なしを保証

### 課題6: プレビューとタイムラインの同期
- **対策:** 再生ヘッドのRatioをもとに `currentSegment(at:)` でアクティブトラックを特定
- **対策:** アクティブトラックが変わった場合のみ `AVPlayer` を切り替え（不要な切り替え防止）

## 10. 参考実装例

### MultipeerConnectivity 接続サンプル
```swift
import MultipeerConnectivity

class CameraSessionManager: NSObject, ObservableObject {
    private let serviceType = "cinecam-sync"
    private let myPeerID = MCPeerID(displayName: UIDevice.current.name)
    
    private var advertiser: MCNearbyServiceAdvertiser?
    private var browser: MCNearbyServiceBrowser?
    private var session: MCSession?
    
    @Published var connectedPeers: [MCPeerID] = []
    @Published var isMaster = false
    
    override init() {
        super.init()
        session = MCSession(peer: myPeerID, 
                           securityIdentity: nil, 
                           encryptionPreference: .required)
        session?.delegate = self
    }
    
    // マスター端末: 録画開始コマンド送信
    func startRecordingAll() {
        let command: [String: Any] = [
            "action": "start_recording",
            "timestamp": Date().timeIntervalSince1970,
            "sessionID": UUID().uuidString
        ]
        sendToAll(data: command)
    }
    
    private func sendToAll(data: [String: Any]) {
        guard let jsonData = try? JSONSerialization.data(withJSONObject: data) else { return }
        try? session?.send(jsonData, toPeers: connectedPeers, with: .reliable)
    }
}
```

## 12. 付録：用語集

| 用語 | 説明 |
|------|------|
| MultipeerConnectivity | Apple提供のP2P通信フレームワーク。AirDropと同じ技術 |
| マスター端末 | 録画開始を制御する主端末 |
| スレーブ端末 | マスターからの指示で動作する従端末 |
| セッションID | 1回の撮影セッションを識別するID |
| timestamp | 録画開始時刻（UNIX時間） |
| P2P通信 | Peer-to-Peer。サーバーを介さない直接通信 |
| アドホックネットワーク | 一時的に構築される端末間ネットワーク |
